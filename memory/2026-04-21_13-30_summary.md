## Key Features Implemented

### 1. Configurable CLI Agents via Environment Variables

**Before:** Hardcoded `AGENTS` object in orchestrator.js

**After:** Environment variable configuration with fallback defaults:

```javascript
const DEFAULT_AGENTS = {
  spec: "claude",
  plan: "claude",
  review: "claude",
  build: "opencode",
  test: "opencode"
};

function getAgentForStage(stage) {
  const envVar = `AGENT_${stage.toUpperCase()}`;
  return process.env[envVar] || DEFAULT_AGENTS[stage];
}
```

**Environment Variables:**
- `AGENT_SPEC` - CLI agent for spec stage (default: claude)
- `AGENT_PLAN` - CLI agent for plan stage (default: claude)
- `AGENT_BUILD` - CLI agent for build stage (default: opencode)
- `AGENT_TEST` - CLI agent for test stage (default: opencode)
- `AGENT_REVIEW` - CLI agent for review stage (default: claude)

### 2. Stage-Specific Skills Compilation

**Before:** Same skills (SKILLS.md, DEBUGGING.md, GIT.md) for all stages

**After:** Different skills per stage based on commands/*.md specifications:

| Stage | Skills |
|-------|--------|
| spec | SKILLS.md, SPEC_DRIVEN.md |
| plan | SKILLS.md, PLANNING.md |
| build | SKILLS.md, INCREMENTAL_IMPLEMENTATION.md, TEST_DRIVEN.md, DEBUGGING.md |
| test | SKILLS.md, TEST_DRIVEN.md, BROWSER_TESTING.md |
| review | SKILLS.md, CODE_REVIEW.md, SECURITY.md, PERFORMANCE.md |

### 3. Failure Analysis Loop

Structured failure analysis instead of blind retries:
```
failure → analyze (Claude) → structured summary → guided fix
```

**Output Format:**
```json
{
  "root_cause": "Missing null check",
  "fix_strategy": "Add guard clause",
  "affected_files": ["src/api.js"],
  "confidence": 0.82
}
```
