# Session Summary — 2026-04-24 10:57

## Overview

This session redesigned how agenticspiq initialises and interacts with workspaces, making it safe to run against existing codebases. Two additional changes were made alongside: agent model names became configurable via environment variables, and all user-facing documentation was rewritten.

---

## 1. Workspace Isolation (`.spiq/` directory)

**Problem:** `utils/init-workspace.sh` created `artifacts/`, `tasks/`, `worktrees/`, `repo/`, and `req.md` at the workspace root, polluting any existing project. There was no mode that avoided this.

**Solution:** All framework state now lives in a hidden `.spiq/` subdirectory inside the workspace. The workspace root is never written to.

```
workspace/.spiq/
├── req.md              ← feature requirements
├── SPEC.md             ← written by spec agent
├── tasks.json          ← pipeline state
├── tasks/plan.md       ← written by plan agent
└── artifacts/
    ├── compiled/       ← compiled stage prompts
    ├── output/         ← agent JSON output
    ├── failures/       ← failure records
    └── logs/
```

### Scaffolding is now automatic and implicit

**Before:** User had to manually run `bash ~/.npm-global/lib/node_modules/agenticspiq/utils/init-workspace.sh`

**After:** `agenticspiq` scaffolds `.spiq/` automatically on first run.

### Requirements sourcing (new UX)

`scaffold.js` handles `req.md` without requiring the user to open `.spiq/`:

1. `--req <file>` flag — copies a specified document
2. Piped stdin — `echo "Build X" | agenticspiq`
3. Auto-detect — looks for `REQUIREMENTS.md`, `SPEC.md`, `BRIEF.md`, `docs/requirements.md`, etc.; prompts user to select
4. Fallback — writes a template to `.spiq/req.md` and exits cleanly for the user to fill in

### New user workflow

```bash
cd /my/existing-project
agenticspiq                          # auto-scaffolds, auto-detects req files
agenticspiq --req docs/brief.md      # explicit requirements source
echo "Build X" | agenticspiq        # stdin
```

## 2. Model Names Configurable

**Problem:** Model names were hardcoded inside agent runner files (`"sonnet"` in `claude.js`, `"opencode/qwen3.5-plus"` in `opencode.js`).

**Solution:** Each runner now reads from an environment variable with a sensible default.

| Env var | Default | Runner |
|---|---|---|
| `CLAUDE_MODEL` | `sonnet` | `agent-cli/runners/claude.js` |
| `OPENCODE_MODEL` | `opencode/qwen3.5-plus` | `agent-cli/runners/opencode.js` |
| `GEMINI_MODEL` | `gemini-2.5-flash-preview` | `agent-cli/runners/gemini.js` (was already configurable) |

Note: the linter also changed the OpenCode flag from `-m` to `--model` during this session.

**Usage:**
```bash
CLAUDE_MODEL=opus agenticspiq                  # one-off override
# or set in .env for persistent configuration
```

`.env` and `.env.example` were both updated with the new vars. `.env.example` was also cleaned up: removed stale `GLM_API_KEY`/`KIMI_API_KEY` entries, aligned section headers and comments with `.env`.
