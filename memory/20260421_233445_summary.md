### 1. Fix dotenv loading when running as global package
**Problem:** `require("dotenv").config()` resolved `.env` relative to `process.cwd()` (the user's workspace directory), which has no `.env`. All `AGENT_*` vars fell back to defaults (`claude`/`opencode`), ignoring the overrides in the package's own `.env`.

**Fix:** Changed `orchestrator.js` line 1 to load dotenv from the package's own directory:
```js
const path = require("path");
require("dotenv").config({ path: path.join(__dirname, "..", ".env") });
```

**Files changed:** `orchestrator/orchestrator.js`

### 6. Fix three issues found in live test run
Issues observed from log: `agenticspiq` running against `multi_agent_test_claude_2`.

#### 6a. Plan stage blocked by Gemini's Plan Mode
**Problem:** `prompts/plan.md` contained "Enter plan mode" which triggered Gemini CLI's own built-in Plan Mode, blocking `write_file` outside its designated plans directory. Result: `tasks/plan.md` never written; build executor found 0 plan matches.

**Fix:** Removed the "Enter plan mode" line from `prompts/plan.md`.

#### 6b. Build executor finds no plan file
**Problem:** Even if the plan agent writes `tasks/plan.md`, any failure leaves the executor blind. The orchestrator should own artifact lifecycle.

**Fix:** Added `writePlanArtifacts(workspace, rawOutput)` in `orchestrator.js` â€” called after the plan stage, writes `tasks/plan.md` from captured stdout output regardless of what the agent did.

#### 6c. Review summary not printed for Gemini
**Problem:** `printReviewSummary` used `JSON.parse(rawOutput).result` (Claude's format). Gemini writes plain text stdout, so the parse failed silently.

**Fix:** Extracted a shared `extractText(rawOutput)` helper that tries JSON parse (`result` â†’ `content` â†’ `text` keys) and falls back to raw string. Both `printReviewSummary` and `writePlanArtifacts` now use it.

**Files changed:** `prompts/plan.md`, `orchestrator/orchestrator.js`
